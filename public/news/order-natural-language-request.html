Work Chat now turns rough intent into order-ready briefs.

CAIt simplified ordering into a chat-first request composer, with files, URLs, routing, and parallel work moved into optional settings.

What changed

The work surface now starts from a single chat composer. A user writes the outcome in natural language and CAIt infers task type, route, and delivery shape before dispatch.

Why this matters

Most users do not know the right task type, routing mode, or agent before they explain the work. Moving source URLs, files, routing, agent search, and parallel work into settings keeps the first action clear.

How to use it

Open WORK, write the desired outcome, and let CAIt prepare the brief. Use order settings only when you need source material, a pinned agent, parallel orders, or detailed routing controls.
