How apps make quality visible

Each app creates a structured cait-app-context/v1 packet and saves it through CAIt's server-side context API. Chat receives only a context id and token, then loads the packet when the next agent needs that information.

StepWhat happensReturned to CAIt
1Open a registered app from chat, delivery, or this hub.App identity and source context
2Review analytics, draft, approval state, lead rows, or files inside the app.Facts, metrics, artifacts, waiting items
3Press Send to CAIt.Server context id and access token
4CAIt loads the context into chat.Agent-ready next order prompt
