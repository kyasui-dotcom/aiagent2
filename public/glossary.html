Core AI Terms
Artificial Intelligence Software techniques that help computers perform tasks that normally require human-like language, perception, reasoning, prediction, or decision-making. Machine Learning A branch of AI where systems learn patterns from data instead of being programmed with every rule by hand. Deep Learning Machine learning that uses multi-layer neural networks to learn complex patterns in text, images, audio, code, and other data. Neural Network A model architecture built from connected layers that transform input data into predictions or generated output. Transformer A neural network architecture that powers many modern language models by paying attention to relationships between tokens. Generative AI AI that creates new output such as text, code, images, audio, video, plans, summaries, or structured reports. Multimodal AI AI that can work across multiple input or output types, such as text, images, audio, video, documents, and code. Natural Language Processing AI techniques for understanding, generating, classifying, translating, summarizing, or extracting information from human language. Computer Vision AI techniques for interpreting images or video, including object detection, OCR, image classification, and visual question answering. OCR Optical character recognition. OCR extracts readable text from images, PDFs, screenshots, scans, and photos. Speech-to-Text AI that transcribes spoken audio into text for search, summarization, subtitles, or voice workflows. Text-to-Speech AI that converts written text into spoken audio for narration, voice assistants, accessibility, or product demos. Token A unit of text processed by a language model. Tokens can be words, word fragments, punctuation, or structured symbols. Training Data The data used to teach a model patterns. Data quality, licensing, bias, and coverage affect model behavior. Dataset A structured collection of examples used for training, evaluation, retrieval, or analysis. Label A target value or annotation used in supervised learning, such as a category, score, answer, or expected output. Supervised Learning Machine learning that uses labeled examples to learn how inputs should map to expected outputs. Unsupervised Learning Machine learning that finds patterns in data without explicit target labels. Reinforcement Learning A learning setup where a system improves behavior by receiving rewards or penalties from actions. Synthetic Data Artificially generated data used for training, testing, privacy protection, or simulation. Model Drift A decline in model quality when real-world inputs change over time compared with the data used to build or evaluate the model. Bias A systematic skew in data, model behavior, or outputs that can create unfair, inaccurate, or incomplete results.
AI Agent Terms
AI Agent Software that uses an AI model to interpret a goal, plan steps, call tools, and produce a result with some level of autonomy. AI Agent Marketplace A platform where AI agents can be discovered, ordered, verified, reviewed, and paid for as reusable software services. AI Agent Runtime The operational layer that handles agent ordering, routing, verification, delivery, logging, cost tracking, retries, and API access. Agent Monetization The process of earning revenue from an AI agent by turning a repeatable capability into a paid, orderable service. Verifiable AI Agent An AI agent with manifest data, readiness checks, endpoints, delivery expectations, and observable behavior that can be checked before use. Agentic AI An AI system designed to act toward a goal, not only answer one prompt. It may plan, use tools, remember context, and check its work. Autonomous Agent An agent that can continue work across multiple steps without asking for human approval at every step. Multi-Agent System A workflow where multiple agents work together, often with different roles such as researcher, coder, reviewer, planner, or verifier. Single-Agent Workflow A task handled by one agent from request to delivery. It is simpler and cheaper when the work does not need multiple specialties. Agent Orchestration The logic that decides which agent should run, in what order, with which inputs, and how outputs are combined. Planner A component that breaks a user request into steps before execution. Planning is useful for long-running research, coding, and analysis work. AI Copilot An AI assistant embedded into a workflow to help a human work faster while the human remains the main decision-maker. Chatbot A conversational interface. A chatbot may only answer messages, while an agent can also take actions and call tools. Human-in-the-Loop A design where the user can review, approve, correct, or add missing information before the agent continues. Memory Stored context that helps an agent remember user preferences, prior work, project facts, or workflow state. Reflection A self-checking step where an agent reviews its own output, plan, or tool results before continuing. Action A step an agent takes outside text generation, such as calling an API, searching the web, writing a file, or creating a pull request.
Model and LLM Terms
LLM A large language model that can read and generate text, code, structured data, and reasoning-like responses based on context. Foundation Model A general-purpose AI model trained on broad data and adapted to many downstream tasks. Prompt The instruction and context given to an AI model. Better prompts define the goal, constraints, examples, output format, and success criteria. Prompt Engineering The practice of improving prompts so the model produces more accurate, useful, and structured outputs. System Prompt High-priority instructions that define assistant or agent behavior, boundaries, style, and safety rules. Context Window The amount of text, files, messages, and tool output a model can consider at once when generating a response. Inference The process of running a model to generate an answer or action from input. Temperature A generation setting that affects randomness. Lower values are more deterministic; higher values can be more creative but less predictable. Top-p A generation setting that limits output choices to a probability mass, controlling creativity and variability. Structured Output Model output constrained to a known format such as JSON, tables, schemas, or specific fields. JSON Mode A mode or instruction pattern that asks the model to return valid JSON for easier parsing by software. Streaming Returning model output incrementally as it is generated, instead of waiting for the full answer. Latency The time between sending a request and receiving a usable response. Fine-Tuning Additional training that adapts a model for a narrower pattern or style. Distillation A technique where a smaller model is trained to imitate a larger model, usually to reduce cost or latency. Model Routing The logic that chooses which model to use for a request based on cost, latency, capability, or quality requirements. Hallucination A confident but incorrect or unsupported model output. Citations, retrieval, verification, and uncertainty reduce this risk.
Retrieval and Knowledge Terms
RAG Retrieval-augmented generation. The system searches trusted documents or data first, then uses the retrieved context to answer. Embedding A numeric representation of text, code, images, or other content used for similarity search and retrieval. Vector Database A database optimized for storing embeddings and finding similar content quickly. Semantic Search Search based on meaning rather than exact keyword match. Grounding Connecting an AI answer to source material, citations, database records, files, or other evidence. Citation A link or reference showing where a claim came from. Corpus The collection of documents, pages, files, or records that a retrieval system can search. Chunking Splitting documents into smaller pieces so retrieval systems can find the most relevant passages for a model. Indexing Preparing content for search by creating keyword indexes, vector indexes, metadata, or lookup structures. Reranking A second search step that reorders retrieved results to put the most relevant passages first. Knowledge Graph A structured map of entities and relationships that can help agents reason over connected facts. Data Connector An integration that gives an AI system access to systems such as Google Drive, GitHub, databases, CRMs, or internal documents.
AI Safety and Security Terms
Prompt Injection An attack where hidden or malicious instructions try to override the system intended behavior. Jailbreak A prompt or technique designed to bypass safety rules, policy constraints, or application guardrails. Data Exfiltration Unauthorized extraction of private data, secrets, source code, or user information from a system. PII Personally identifiable information, such as names, emails, phone numbers, account IDs, addresses, or government identifiers. Secret A credential such as an API key, token, webhook signing secret, database password, or OAuth client secret. Least Privilege A security principle where users, agents, and integrations receive only the permissions needed for their task. Sandbox An isolated environment for running code or agent actions with limits that reduce the blast radius of mistakes or attacks. Content Filtering Rules or classifiers that block, flag, or transform unsafe, irrelevant, or policy-violating content. Audit Log A record of important actions, requests, billing events, security decisions, and operational changes. Data Retention The policy that determines how long user data, prompts, files, logs, and outputs are stored. Guardrail A rule, validator, permission boundary, or review step that keeps an AI system within safe and expected behavior.
AI Evaluation Terms
Benchmark A standard task or dataset used to compare model or agent performance. Eval A test that checks whether an AI system produces correct, safe, useful, and properly formatted results. Regression Test A test that ensures a new change does not break behavior that previously worked. Golden Dataset A trusted set of example inputs and expected outputs used to evaluate quality over time. Ground Truth The expected correct answer or trusted reference used to judge AI output. Precision The share of returned results that are relevant or correct. Recall The share of relevant results that the system successfully finds. Confidence A signal that describes how reliable a model, agent, or delivery appears to be. Red Teaming Testing an AI system by intentionally trying to make it fail, leak data, ignore rules, or produce unsafe output. Observability Logs, traces, metrics, and dashboards used to understand how an AI system behaves in production.
AI Operations Terms
Tool Calling The model asks the runtime to call an external tool, API, database, browser, or function instead of only producing text. Function Calling A structured form of tool calling where the model returns arguments for a defined function. MCP Model Context Protocol. A standard pattern for exposing tools and context sources to AI applications. Workflow Automation Software automation that runs repeated steps. An AI agent can be part of a workflow, but workflows can also be deterministic. API An application programming interface used by software systems to communicate. Webhook An HTTP callback sent from one system to another when an event happens. OAuth A login and authorization protocol used to connect accounts such as Google or GitHub without sharing passwords with CAIt. API Key A token used by CLI or server-side clients to authenticate API requests. It should be stored like a secret. CLI A command-line interface for running workflows from a terminal instead of a browser. SDK A software development kit that provides client libraries, helpers, and examples for building against an API. Rate Limit A limit on how many requests can be made in a time window to protect reliability, cost, and abuse resistance. Queue A system that stores work until a worker or agent is ready to process it. Retry Running a failed or timed-out action again, usually with limits to avoid loops or duplicate side effects. Timeout The maximum time a system waits before stopping a request or marking it as failed. Idempotency A property where repeating the same request does not create unintended duplicate effects. Webhook Signature A cryptographic check that confirms a webhook came from the expected service and was not modified in transit.
CAIt Terms
Order A request submitted to CAIt. The user describes the outcome, and CAIt routes the work to a suitable agent. Delivery The result returned by an agent. A good delivery includes an answer, files, sources, cost, confidence, and next actions. Built-In Agent An agent provided inside CAIt so users can try the product before publishing their own agent. Manifest A machine-readable description of an agent, including name, owner, task types, endpoint, capabilities, and verification metadata. Verification The readiness check that confirms an agent has the required manifest, health endpoint, job endpoint, and expected response behavior. Auto-Routing The default routing mode where CAIt infers the task and chooses a ready agent instead of requiring manual selection. Broker The routing layer that matches an order to a suitable agent based on task type, readiness, and fit. Adapter PR A GitHub pull request generated by CAIt that adds files for an existing app to expose CAIt-compatible endpoints. Month-End Billing Saved-card billing model where CAIt records settled order usage and charges after the billing period. Provider Payout Money earned by an agent provider and withdrawn through Stripe Connect after eligible work is completed. CAIt API Key A user-scoped API key used to create and read orders, register agents, verify owned agents, and request GitHub adapter PR creation from CLI or server-side clients. Repository Write Confirmation An explicit confirmation required before CAIt creates a GitHub adapter pull request from an API-key client.
