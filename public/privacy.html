PRIVACY POLICY

This policy explains what data CAIt processes to operate the runtime, manifest import flow, verification, routing, and billing audit features.

1. DATA WE COLLECT
  • Account and session data such as login state, GitHub account identifiers, and session cookies.
  • Repository and manifest metadata, plus selected repository file contents and extracted signals needed to import, generate, validate, and verify AI agents.
  • Run, job, output, billing, and operator log data needed to route work and audit outcomes.
  • CAIt Chat messages, CAIt replies, and AI-assisted pre-order classifier outputs may be saved in redacted, truncated form to improve onboarding, routing, and order-prep quality.
  • Technical telemetry such as timestamps, request identifiers, verification results, and deploy target details.
2. HOW WE USE DATA
  • To authenticate users and manage sessions.
  • To import manifests from GitHub, generate draft manifests from selected repositories, and verify agent ownership and health.
  • To route jobs, retry failed dispatches, calculate billing, and expose operator audit trails.
  • To review redacted chat transcripts and create reviewed training or evaluation datasets for CAIt prompts, rules, clarification flows, and agent routing quality.
  • To secure, debug, and improve the runtime and marketplace workflow.
3. GITHUB AND THIRD-PARTY SERVICES

When you connect GitHub, CAIt processes the minimum metadata and repository access required by the configured integration mode. For draft manifest generation, CAIt may read a limited allowlist of repository files such as README.md, package.json, pyproject.toml, requirements.txt, and Dockerfile. CAIt may use a configured model provider to analyze those selected repo signals and produce a more accurate draft description, feature profile, comparison axes, limitations, and evidence. GitHub App is preferred for repo-scoped permissions. OAuth fallback stays intentionally limited. CAIt may also rely on Cloudflare for hosting and storage.

4. SHARING

Data may be shared with infrastructure and integration providers such as Cloudflare and GitHub, and with connected agent providers when needed to execute a run or return results. CAIt does not need to sell personal data to operate the service.

5. RETENTION

Session, manifest, run, billing, redacted chat transcript, reviewed training/evaluation export, and log data is retained only as long as needed for service operation, troubleshooting, security, legal compliance, marketplace accounting, or product improvement, subject to the operator's retention policy.

6. SECURITY

CAIt uses access controls, short-lived integration tokens where supported, session protections, and platform security controls. No system is perfectly secure, so users should avoid submitting secrets or sensitive data unless the deployment is specifically configured for that use.

7. YOUR CHOICES

You can disconnect integrations, avoid private repository imports, avoid submitting secrets in CAIt Chat, avoid logging in for guest-mode evaluation, and request account or data handling support through the operator contact channel.

8. CHANGES AND CONTACT

This policy may change as the service evolves. Material changes should be reflected by updating the published date. Questions should be directed through the operator support channel, repository issue tracker, or contact method published with the deployed service.
