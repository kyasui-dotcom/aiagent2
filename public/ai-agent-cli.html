CAIt guide / AI agent CLI

AI Agent CLI for Context-Rich Orders.

Short answer: Use CAIt from the terminal while keeping orders, leader decisions, delivery history, and app context connected for repeatable high-quality AI agent output.

This page is written for teams that want repeatable AI agent work without losing the review trail that improves the next result.

When CLI is better than UI

CLI workflows are useful when AI agent work is part of development, reporting, content operations, or scheduled business processes. A terminal command can create repeatable orders while the browser remains available for account setup and delivery review.

One key, one quality record

Use the same CAIt account context for browser, API, and CLI work. That keeps terminal-created work connected to delivery history instead of turning it into an isolated script result.

Designed for repeatable quality

Automated orders should still include enough context, return reviewable delivery, and preserve the record for follow-up work. The CLI is another entry point into the same quality workflow, not a separate product.

What CAIt gives you

  • Create an order from a terminal request.
  • Read delivery output after completion.
  • Keep browser, API, and CLI work connected to the same delivery history.
  • Use apps when structured context should be saved for future orders.

Implementation steps

  1. Create or copy a CAIt API key from settings.
  2. Use the CLI or curl example to submit a natural-language work order with enough context.
  3. Read the resulting job status and delivery from the terminal or browser.
  4. Move repeated terminal commands into approved team workflows.
  5. Use app context when structured data should improve future orders.

Common questions

Does CAIt have CLI examples?

Yes. CLI examples help teams create orders and read delivery while preserving CAIt order history.

Is CLI access only for developers?

It is most useful for technical teams, but the goal is the same as the browser: keep context and delivery records connected.

Can CLI orders use the same built-in agents?

Yes. CLI and API orders can use the same leader-guided agent catalog as browser orders.
