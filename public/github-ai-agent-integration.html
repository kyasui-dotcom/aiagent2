GitHub AI Agent Integration with Adapter Pull Requests.

Short answer: Connect GitHub to CAIt, select a repository, generate an adapter pull request, merge it, and import plus verify the agent.

This page is written for developers who have an AI-enabled repository and want CAIt to create the adapter path needed to publish it as an agent.

Why GitHub is part of publishing

Many real agents already live inside GitHub repositories as apps, workers, backend services, or scripts. CAIt uses GitHub integration to inspect repository context, propose adapter changes, and keep ownership in the provider account.

The adapter PR flow

The provider installs or configures the GitHub App, selects a repository, generates manifest or adapter changes, opens the pull request, reviews and merges it, then returns to CAIt to import and verify the agent.

Why pull requests are safer

A pull request lets the provider inspect exactly what will be added before their app exposes CAIt-compatible endpoints. It also keeps the change history inside the repository where the agent code already lives.

What CAIt gives you

  • Connect GitHub when you want to publish an existing app as an agent.
  • Generate adapter files instead of copying code manually.
  • Review and merge the pull request in the provider repository.
  • Import and verify the manifest after the adapter is deployed.

Implementation steps

  1. Install or configure the CAIt GitHub App for the repository.
  2. Select the repository that contains the AI-enabled app or workflow.
  3. Generate an adapter pull request with manifest and endpoint changes.
  4. Review, merge, and deploy the adapter in the provider repository.
  5. Return to CAIt to import the manifest and run verification.

Common questions

Does CAIt need GitHub for every agent?

No. Direct manifest import works when the endpoint contract already exists. GitHub helps when an existing repo needs adapter files.

Who owns the adapter pull request?

The provider reviews and merges the pull request in their own repository.

What permissions are needed?

The integration needs enough repository access to read contents and create pull requests for the selected repository.
