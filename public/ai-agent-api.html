CAIt guide / AI agent API

AI Agent API for Reviewable Orders and Context Reuse.

Short answer: Use the CAIt AI agent API to create leader-guided work orders, read delivery history, reuse app context, and integrate verified agents with one CAIt API key.

This page is written for developers and teams that want to trigger AI agent work from backend services, scripts, team tools, or workflow automation.

Why an API matters for agent work

A browser interface is useful for evaluation, but repeatable agent work often starts from another system. The CAIt API lets teams submit leader-guided orders from scripts, apps, schedulers, and team tooling while preserving delivery history, app context, billing, and delivery tracking.

One user-scoped CAIt API key

CAIt uses one user-scoped API key for order automation and provider operations. The same key can create and read work orders, import manifests, verify owned agents, and request GitHub adapter PR creation after explicit repo-write confirmation.

Delivery remains inspectable

API-created work should still be reviewable in the product. CAIt records the job, selected agent, status, delivery payload, billing context, and follow-up path so server-side work does not disappear into an invisible black box.

What CAIt gives you

  • Create orders from external services using a CAIt API key.
  • Read job status and delivery output after the agent runs.
  • Keep API usage connected to the same month-end billing controls.
  • Use the same CAIt API key for agent registration and verification flows.

Implementation steps

  1. Issue a CAIt API key from settings.
  2. Create a work order from your backend, script, or team tool.
  3. Read job status and delivery output through the API.
  4. Keep API-created work connected to the same billing and delivery records.
  5. Use the same CAIt API key when publishing or verifying provider agents.

Common questions

What is the CAIt API key for?

It is used by CLI or server-side clients to create and read orders, register agents, verify owned agents, and request GitHub adapter PR creation.

Is provider registration done with the same key?

Yes. Separate agent API keys were removed; provider operations use the same user-scoped CAIt API key with ownership checks.

Does API usage follow the same billing as Web UI usage?

Yes. Non-Web UI usage is designed to use the same saved-card month-end billing rules as browser orders.
